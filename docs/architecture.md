# 系统架构文档

## 概述

热点信息自动收集与处理系统采用模块化架构设计，由多个独立的组件协同工作，实现每日热点信息的自动收集、分析、总结和分发。

## 核心组件

### 1. 工作流调度器 (WorkflowScheduler)

**职责**: 协调整个工作流程，管理定时任务执行。

**主要功能**:
- 定时触发信息收集任务
- 编排各模块的执行顺序
- 监控任务执行状态
- 处理异常和恢复

**关键特性**:
- 支持cron表达式配置
- 错误重试机制
- 状态持久化

### 2. 爬虫管理器 (CrawlerManager)

**职责**: 管理和协调多个信息源爬虫。

**主要功能**:
- 加载和配置爬虫实例
- 并行执行多个爬虫
- 频率控制和访问限制
- 错误隔离和容错

**支持的爬虫类型**:
- RSS源爬虫 (RSSCrawler)
- 微博热搜爬虫 (WeiboCrawler)
- 知乎热榜爬虫 (ZhihuCrawler)
- 新闻网站爬虫 (NewsCrawler)

### 3. 内容分析模块 (Analyzers)

**职责**: 对收集的原始内容进行智能分析和处理。

**主要功能**:
- 文本分词和清洗
- 关键词提取 (TF-IDF, TextRank)
- 情感分析
- 新闻分类
- 热度计算

**算法支持**:
- jieba中文分词
- TF-IDF关键词提取
- TextRank算法
- LDA主题模型

### 4. 总结生成模块 (Summarizers)

**职责**: 将分析结果转换为公众号格式的总结。

**主要功能**:
- HTML模板渲染
- 内容格式化
- 排版优化
- 多格式输出 (HTML, 文本)

**模板引擎**:
- Jinja2模板引擎
- 自定义过滤器
- 响应式设计

### 5. 邮件发送模块 (Mailers)

**职责**: 将生成的总结发送到指定邮箱。

**主要功能**:
- SMTP邮件发送
- HTML邮件支持
- 附件支持
- 发送失败重试

### 6. 配置管理模块 (Config)

**职责**: 管理系统配置和环境变量。

**主要功能**:
- YAML配置文件加载
- 环境变量管理
- 配置验证
- 敏感信息保护

## 数据流

```
1. 定时触发 (每日19:00)
      ↓
2. 爬虫管理器并行执行
      ↓
3. 收集原始新闻数据
      ↓
4. 内容分析和处理
   - 分词
   - 关键词提取
   - 情感分析
   - 分类
   - 热度计算
      ↓
5. 过滤和排序
      ↓
6. 生成总结 (HTML/文本)
      ↓
7. 发送邮件
      ↓
8. 记录日志
```

## 扩展点

### 添加新信息源

1. 继承 `BaseCrawler` 基类
2. 实现 `fetch()` 和 `parse()` 方法
3. 在配置文件中添加信息源配置
4. 在爬虫管理器中注册新爬虫

### 自定义分析算法

1. 继承或扩展现有分析器
2. 实现自定义分析逻辑
3. 在配置中指定使用的分析器

### 自定义总结模板

1. 在 `config/templates/` 目录下创建HTML模板
2. 使用Jinja2模板语法
3. 在配置中指定模板文件名

### 添加新的分发渠道

1. 创建新的发送器类
2. 实现发送接口
3. 在工作流中添加分发步骤

## 技术栈

- **Python**: 3.8+
- **爬虫**: requests, beautifulsoup4, feedparser
- **分析**: jieba, scikit-learn
- **模板**: Jinja2
- **调度**: APScheduler
- **配置**: PyYAML, pydantic
- **日志**: loguru
- **测试**: pytest
